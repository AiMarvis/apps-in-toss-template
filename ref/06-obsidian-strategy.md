# 🏭 앱인토스 템플릿 전략 — 반복 가능한 앱 공장

> **한 줄 요약**: Moodi/Soom/실제 반려사례 DB에서 검증된 코드·문서·운영 패턴을 템플릿화하여, 새 앱을 **1주일 이내** MVP 출시 + 검수 반려 최소화까지 달성하는 전략

---

## 📊 현황 분석: 3개 앱의 공통 패턴

| 앱 | 상태 | 고유 기능 | 공통 기반 |
|:---|:---|:---|:---|
| **🎵 Moodi** | 검수 중 | AI 음악 생성 | Granite + Firebase Auth + Convex + Zustand |
| **📖 BookStreak** | 리서치 완료 | 독서 스트릭/통계 | 〃 (Moodi 70% 재사용) |
| **📈 집단지성** | 설계 완료 | 실시간 투표/AI 대결 | 〃 + Cron Jobs + 실시간 |

### 공통 코드 비율: ~60~70%

```
📦 전체 앱 구성
├── 🔵 공통 (템플릿) ─── 약 60~70%
│   ├── Firebase Auth + 토스 로그인
│   ├── Convex 연결 설정
│   ├── 백버튼 핸들링 (closeView)
│   ├── 연동 해제 감지
│   ├── TDS 컴포넌트 래퍼
│   ├── 법적 페이지 (약관, 개인정보, 면책)
│   ├── 상태관리 기본 구조 (Zustand)
│   └── 프로젝트 설정 (granite.config, convex 초기화)
│
└── 🟠 앱 고유 ─────── 약 30~40%
    ├── DB 스키마 (앱마다 다른 테이블)
    ├── Convex 함수 (앱 비즈니스 로직)
    ├── UI 컴포넌트 (앱 화면)
    └── 외부 API 연동 (앱마다 다른 API)
```

## 🎯 4-Layer 접근법

```
┌─────────────────────────────────────────────┐
│  Layer 4: 검수 운영 시스템                    │
│  실기기 테스트 매트릭스 + 제출 게이트          │
├─────────────────────────────────────────────┤
│  Layer 3: AI 개발 룰                         │
│  AGENTS.md + .cursorrules                   │
│  (AI가 검수 기준을 자동으로 적용)              │
├─────────────────────────────────────────────┤
│  Layer 2: 문서 템플릿                         │
│  Plan / Design / 체크리스트 템플릿             │
│  (Obsidian에서 새 앱 기획 빠르게 시작)          │
├─────────────────────────────────────────────┤
│  Layer 1: 코드 보일러플레이트                   │
│  GitHub Template Repo                       │
│  (npx create → 즉시 개발 시작)               │
└─────────────────────────────────────────────┘
```

## 📈 템플릿 효과 예측

| 항목 | 템플릿 없이 | 템플릿 사용 |
|------|-----------|-----------|
| **MVP 개발 기간** | 2~4주 (Moodi 기준) | **3~7일** |
| **검수 반려 횟수** | 다회 반복 | **0~1회 목표** |
| **코드 작성량** | 100% | **30~40%** (고유 부분만) |
| **검수 대응 시간** | 수일 (원인 파악부터) | **수시간** (패턴 숙지) |

## 🚀 새 앱 시작 워크플로우

### Step 1: 리서치 (1일)
1. NotebookLM 노트북 생성
2. 시장 분석 + 경쟁 앱 조사
3. NotebookLM에 분석 결과 저장

### Step 2: 기획 (1일)
1. Obsidian에 새 폴더 생성
2. Plan 문서 템플릿 복사 → 작성
3. 기술 스택은 공통 기반 + 앱 고유 API만 추가

### Step 3: 설계 (1일)
1. Design 문서 템플릿 복사 → 작성
2. UI/UX 화면 설계 (앱 고유 부분만)
3. Convex 스키마 정의 (앱 고유 테이블만)
4. 토스 검수 체크리스트 확인

### Step 4: 개발 (3~5일)
1. 템플릿 레포 복제
2. 환경 변수 설정
3. 앱 고유 코드만 작성 (나머지는 템플릿 그대로!)

### Step 5: 배포 & 검수 (1일)
1. Convex 배포 + 프론트엔드 빌드 & 업로드
2. 체크리스트 전체 확인
3. Layer 4 게이트 실행 → 모두 pass 후 제출
