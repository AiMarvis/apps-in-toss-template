Toss 네이티브 로그인 → Firebase Custom Token → Convex JWT 검증의 전체 인증 체인

🔐 인증 체인 (7단계)

1
온보딩 화면 표시

↓
2

tossAuth.ts → SDK의 appLogin()으로 토스 네이티브 로그인 UI 실행
↓
3a
직접 Token 반환
customToken이 바로 반환되는 경우
OR
3b
Authorization Code 교환
Cloud Function (mTLS) → Firebase Custom Token
↓
4
Firebase signInWithCustomToken()

↓
5

useConvexAuth.ts — onIdTokenChanged 감지 → getIdToken()으로 JWT 획득 → Convex에 전달
↓
6
Convex JWT 검증
convex/auth.config.ts — Firebase Auth 도메인 기반으로 JWT 유효성 확인
↓
7
Convex 유저 레코드 생성/갱신
convex/users.ts → getOrCreate 뮤테이션으로 upsert (firebaseUid 기준)

🛡️ 연동 해제 감지 (3-Point Re-verification)

토스 심사 필수 — 사용자가 토스 앱에서 연동 해제 시 즉시 로그아웃 처리

1

앱 시작

authStore.initialize()
onAuthStateChanged에서 guardDisconnect() 호출

2

Auth 상태 변경

guardDisconnect()에서
getIsTossLoginIntegratedService() 호출
isLinked === false → 로그아웃

3

포그라운드 복귀

useTossIntegration.ts
visibilitychange 이벤트 리스너로
복귀 시 연동 상태 재확인

⚠️
주의: 반드시 isLinked === false (strict comparison)로 비교해야 합니다. !isLinked는 undefined나 null도 true로 평가하여 오작동을 유발합니다.

SDK 안전 패턴

토스 SDK는 반드시 동적 import + isSupported 체크 패턴을 따라야 합니다

🔒 3단계 안전 패턴 sdkSafety.ts

❌ 잘못된 방법
// Static import → 웹 환경에서 크래시 import { someAPI } from '@apps-in-toss/web-framework'; someAPI(); // 💥 번들 크기 증가 + 크래시
✅ 올바른 방법
// Step 1: Dynamic import const { someAPI } = await import( '@apps-in-toss/web-framework' ); // Step 2: isSupported 체크 if (someAPI.isSupported() !== true) { return; // mock 동작 } // Step 3: 안전하게 호출 someAPI();

유틸리티 함수

함수설명반환값
sdkImportAndCheck(apiName) 동적 import + isSupported 체크를 한 번에 API 함수 또는 null
callSdkWithCleanup(apiName, opts) SDK 호출 + cleanup 함수 반환 (useEffect용) cleanup () => void
