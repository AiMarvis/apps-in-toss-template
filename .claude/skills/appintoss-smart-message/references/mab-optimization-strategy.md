# MAB 최적화 전략 — 스마트 발송 알고리즘 이해하기

스마트 발송의 핵심은 **Multi-Armed Bandit(MAB) 알고리즘**이다. 이 알고리즘을 이해하면 소재 세트를 전략적으로 설계해서 CTR을 극대화할 수 있다.

---

## MAB 알고리즘이란

스마트 발송은 여러 소재 중 CTR이 높은 것에 자동으로 노출을 집중하는 시스템이다. 내부적으로 MAB 알고리즘이 작동한다.

### 핵심 원리: 탐색(Exploration) vs 활용(Exploitation)

1. **탐색(Exploration)**: 모든 소재를 골고루 노출해서 각각의 실제 CTR을 파악
2. **활용(Exploitation)**: CTR이 높은 소재에 더 많은 노출을 집중

캠페인 초기에는 탐색이 많고, 시간이 지날수록 활용이 늘어난다. 최종적으로 승자 소재가 가장 많은 노출을 받는다.

### A/B 테스트와의 차이

| A/B 테스트 | MAB (스마트 발송) |
|-----------|-----------------|
| 50:50 고정 비율로 노출 | 성과에 따라 동적 배분 |
| 테스트 기간 동안 패배 소재에도 균등 노출 | 패배 소재의 노출을 빠르게 줄임 |
| 테스트 종료 후 승자 선택 | 실시간으로 승자에 집중 |
| 기회비용이 큼 | 기회비용 최소화 |

## 소재 개수의 전략적 의미

### 왜 5개 이상이어야 하는가

- **2~3개**: 알고리즘이 탐색에 많은 노출을 "낭비". 승자를 찾아도 대안이 적어 최적화 한계
- **5~7개**: 다양한 가설을 빠르게 검증. 하나가 압도적 성과를 보이면 집중
- **7~10개**: 가장 이상적. 다양한 앵글에서 승자를 찾고, MAB가 빠르게 수렴
- **10개 이상**: 탐색 기간이 길어질 수 있으나, 최종 승자의 CTR이 가장 높음

**권장: 최소 5개, 이상적으로 7~10개**

### 소재 간 차별화가 핵심

MAB 알고리즘은 소재 간 CTR **차이**가 클수록 빨리 승자를 찾는다.

**나쁜 세트 (차별화 없음):**
```
#1: "습관 체크 9원" / "토스에서 9원 받아요."
#2: "습관 1번 3원" / "토스에서 3원 받아요."
#3: "습관 완료 9원" / "토스에서 9원 받아요."
→ 3개 모두 "보상" 앵글. CTR 차이 미미 → MAB가 구분 불가 → 무작위 배분
```

**좋은 세트 (5앵글 활용):**
```
#1 [호기심]: "판다가 배고프대요" / "토스에서 판다에게 밥 줘보세요."
#2 [행동묘사]: "버튼 3번이면 끝" / "토스에서 습관 체크해 보세요."
#3 [확실한보상]: "매일 9원 확정" / "토스에서 체크만 하면 돼요."
#4 [감성]: "판다가 '맛있어~'" / "토스에서 대나무 줘보세요."
#5 [루틴]: "아침 체크 한 번이면" / "토스에서 3원 받고 시작해요."
→ 5개 각각 다른 앵글. CTR 차이 확실 → MAB가 빠르게 승자 선택
```

## 캠페인 기간과 학습 속도

### 너무 짧으면 탐색만 하다 끝남

- **1~3일**: 탐색만 하다 종료. 데이터 부족으로 최적화 불완전
- **7~14일**: 탐색 → 활용 전환이 일어남. 승자가 드러나기 시작
- **14~28일**: MAB가 충분히 학습 → 승자 소재에 집중 → 최적 CTR
- **28일+**: 소재 피로도 발생 가능. 새 소재로 교체 권장

**권장: 2~4주 단위로 캠페인 운영, 소재 교체**

### 검토 기간 역산

소재 검토에 영업일 2~3일 소요되므로:
- 캠페인 종료 1주 전에 다음 소재 세트를 제출
- 검토 반려 시 수정 → 재제출 기간 확보

## Winner/Loser 복기 프레임워크

캠페인 종료 후 (또는 7일 경과 후) 반드시 성과를 복기한다.

### 복기 질문

1. **Winner 분석**
   - 어떤 앵글이었나? (호기심/행동/보상/감성/루틴)
   - 핵심 키워드가 뭐였나? (캐릭터? 행동? 금액?)
   - 타이틀↔본문 연결이 어떤 공식이었나? (보상→행동? 호기심→행동?)
   - 왜 이겼다고 생각하나? (가설)

2. **Loser 분석**
   - 어떤 앵글이었나?
   - 왜 안 됐다고 생각하나?
   - 데이터로 확인되는 패턴이 있나? (노출은 많았는데 클릭이 없었다 vs 노출 자체가 적었다)

3. **다음 세트 설계**
   - Winner의 핵심 키워드를 유지하되, 앵글을 교차한다
   - Loser의 패턴은 명시적으로 피한다
   - 새로운 가설 1~2개를 추가한다

### 복기 템플릿

```
[캠페인 성과 복기]
캠페인명: ___
기간: ___ ~ ___
소재 수: ___ 개

소재별 성과:
| # | 앵글 | 타이틀 | CTR | 노출수 | 판정 |
|---|------|-------|-----|--------|------|
| 1 | ___ | ___ | ___% | ___ | Winner/Loser |
| 2 | ___ | ___ | ___% | ___ | Winner/Loser |
| ... |

Winner 인사이트: ___
Loser 인사이트: ___
다음 세트 방향: ___
```

## 실전 팁

### 팁 1: "안전한 소재"와 "실험 소재"를 섞어라

- 안전한 소재 3~4개: 검증된 패턴 (보상 앵글, A/B 검증된 키워드)
- 실험 소재 2~3개: 새로운 앵글, 새로운 키워드

MAB가 안전한 소재를 기본으로 놓고, 실험 소재가 이기면 새 승자를 발견한다.

### 팁 2: 노출수가 극단적으로 적은 소재는 패배가 아니다

MAB는 초기에 소재를 탐색할 때, **어떤 소재에 노출을 거의 안 줄 수도 있다**. 이것은 그 소재가 나쁘다는 뜻이 아니라, 알고리즘이 다른 소재를 우선 탐색한 것이다. 노출수 100 미만인 소재의 CTR은 참고만 한다.

### 팁 3: 금액 미명시 소재는 피하라

A/B 데이터에서 "포인트 받기" (금액 미명시) 소재는 노출 자체가 291건밖에 안 됐다. MAB 알고리즘이 초기 탐색에서 이 소재의 CTR이 낮다고 판단하면, 이후 노출을 거의 주지 않는다. **금액 미명시 = 노출 기회 상실**.
