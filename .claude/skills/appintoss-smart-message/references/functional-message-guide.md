# 기능성 메시지 가이드

앱인토스 스마트 발송에는 **광고성 메시지**와 **기능성 메시지** 두 종류가 있다. 이 가이드는 기능성 메시지의 작성 규칙과 광고성 메시지와의 시너지 전략을 다룬다.

---

## 기능성 vs 광고성 구분

### 광고성 메시지
- **목적**: 상품/서비스를 알리고, 특정 행동을 유도
- **운영 방식**: 콘솔에서 소재 등록 → MAB 자동 최적화
- **발송**: 시스템이 최적 시간에 자동 발송
- **검토**: 소재 검토 필수 (영업일 2~3일)
- **예**: "심리 테스트로 나의 연애 유형을 알아보세요."

### 기능성 메시지
- **목적**: 서비스 이용에 필요한 정보 전달 (거래확인, 완료통보 등)
- **운영 방식**: API로 서버에서 자동 발송 (templateSetCode 기반)
- **발송**: 유저 행동 트리거 시 즉시 발송
- **검토**: 템플릿 검토 필수
- **예**: "요청하신 이미지 생성이 완료됐어요."

## 기능성 메시지 작성 규칙

### ✅ 허용되는 내용

**서비스 이용에 직접적으로 필요한 정보만**:

| 유형 | 예시 |
|------|------|
| 거래 관련 | "결제 완료", "환불 처리됨", "포인트 적립됨" |
| 진행 상황 | "배송 시작", "이미지 생성 완료", "예약 확정" |
| 기간/상태 알림 | "정기 결제 예정", "이용권 만료 7일 전" |

### ❌ 금지되는 내용

**광고·판촌·유도성 메시지는 기능성에서 반려됨**:

| 반려 패턴 | 이유 | 올바른 분류 |
|----------|------|-----------|
| "이런 상품은 어떠세요?" | 판촉 목적 | 광고성으로 보내야 함 |
| "오늘만 할인 중이에요." | 판촉 목적 | 광고성으로 보내야 함 |
| "이벤트 참여하고 혜택 받아가세요." | 이벤트 홍보 | 광고성으로 보내야 함 |
| "곧 오늘의 보상이 사라져요." | 긴급성 유도 | 광고성으로 보내야 함 |
| "명상할 시간이에요. 인증하면 포인트" | 행동 유도 | 광고성으로 보내야 함 |
| "점심 메뉴를 추천해 드려요." | 서비스 추천 | 광고성으로 보내야 함 |
| "토스에서 선물을 확인해 보세요." | 행동 유도 | 광고성으로 보내야 함 |

### 🔍 경계선 사례

이런 경우 기능성인지 광고성인지 애매할 수 있다:

| 메시지 | 판단 | 이유 |
|--------|------|------|
| "주문이 완료되었어요." | ✅ 기능성 | 거래 완료 알림 |
| "주문 완료! 다음 주문 시 10% 할인" | ❌ 반려 | 기능성 + 광고 혼합 |
| "이용권 만료 7일 전이에요." | ✅ 기능성 | 상태 알림 |
| "이용권 만료 전에 갱신하면 30% 할인" | ❌ 반려 | 기능성 + 광고 혼합 |
| "요청하신 이미지 생성이 완료됐어요." | ✅ 기능성 | 진행 완료 알림 |
| "이미지 완성! 친구에게 공유해보세요." | ❌ 반려 | 기능성 + 행동 유도 |

**원칙: 기능성 메시지는 "순수 정보 전달"만. 행동 유도, 보상 언급, 판촉 표현이 1글자라도 들어가면 광고성으로 분류해야 한다.**

## 기능성 메시지 API 기본

### 발송 방식

기능성 메시지는 서버(API)에서 자동 발송한다:

```
POST /api-partner/v1/apps-in-toss/messenger/send-message
Header: X-Toss-User-Key: {유저 정보 조회에서 나온 userKey}
Body: {
  "templateSetCode": "template_001",
  "context": {
    "storeName": "내앱이름",
    "date": "2025-01-20 15:30"
    // userName은 자동으로 유저 이름 적용 (전달 불필요)
  }
}
```

### 주요 사항

- `templateSetCode`: 콘솔에서 등록한 템플릿 코드
- `context`: 템플릿 내 변수에 들어갈 값 (동적 데이터)
- `userName`: 자동으로 유저 이름이 적용되므로 전달할 필요 없음
- 테스트 발송 API로 실제 발송 전 검증 가능

## 광고성 + 기능성 시너지 전략

### 두 채널의 역할 분담

```
[유저 라이프사이클에서의 역할]

광고성 메시지: 유저 획득 + 재방문 유도
├─ 신규 유저에게 서비스 소개 (MAB 최적화)
├─ 이탈 유저에게 밀린 보상 알림
└─ 시즌 이벤트 프로모션

기능성 메시지: 활성 유저 유지 + 행동 완료 알림
├─ 습관 체크 완료 알림
├─ 미션 달성 알림
├─ 기간 만료 사전 알림
└─ 거래 완료 확인
```

### 시너지 플로우 예시

**습관 체크 앱 시나리오:**

```
1. [광고성] "매일 9원 확정" → 신규 유저 앱 진입
2. [기능성] "오늘의 습관 체크가 완료됐어요." → 앱 내 행동 후 알림
3. [기능성] "이번 주 미션을 모두 완료했어요." → 주간 완료 확인
4. [광고성] "밀린 보상 27원" → 7일 미접속 유저 재방문 유도
5. [기능성] "이용권 만료 7일 전이에요." → 갱신 사전 알림
```

### 주의사항

1. **기능성 메시지에 광고를 절대 섞지 않는다** — 반려 1순위
2. **같은 유저에게 광고성+기능성이 너무 많이 가지 않도록** 빈도 관리
3. **기능성 메시지의 톤도 토스 톤을 유지** — "~요." 해요체, 친절하게
4. **기능성 메시지 검토도 별도로 받아야 한다** — 템플릿 코드 기반
