# 세그먼트 타겟팅 가이드

앱인토스 스마트 발송에서는 **세그먼트(유저 그룹)**를 설정해서 특정 유저 집단에게만 메시지를 보낼 수 있다. 같은 앱이라도 유저 상태에 따라 메시지 전략이 완전히 달라야 한다.

---

## 세그먼트란

세그먼트는 앱인토스 콘솔의 좌측 메뉴 '세그먼트'에서 만드는 유저 그룹이다. 캠페인 생성 시 '타겟' 항목에서 세그먼트를 선택하면, 해당 그룹에만 메시지가 발송된다.

- **설정 경로**: 앱인토스 콘솔 → 워크스페이스 → 미니앱 → '세그먼트' 메뉴
- **상세 가이드**: [세그먼트 콘솔 가이드](https://developers-apps-in-toss.toss.im/segment/console.html)

---

## 핵심 세그먼트 분류

### 1. 전체 유저 (미접속 포함)

**특성**: 앱을 한 번도 열어본 적 없는 토스 유저. 내 앱에 대해 아무것도 모름.
**핵심 전략**: 발견 + 호기심
**소재 앵글**: 호기심, 확실한 보상

| 타이틀 | 본문 | 앵글 |
|--------|------|------|
| "성격 테스트" | "토스에서 바로 써볼 수 있어요." | 서비스 발견 |
| "내 연애 유형" | "토스에서 5분이면 알 수 있어요." | 호기심 |
| "9원 게임" | "토스에서 바로 도전해보세요." | 확실한 보상 |

### 2. 1회 방문 후 이탈

**특성**: 앱을 열어봤지만 핵심 행동을 완료하지 않음. 관심은 있었지만 빠짐.
**핵심 전략**: 미완료 유도 + 미수령 보상 알림
**소재 앵글**: 미수령, 감성·재미

| 타이틀 | 본문 | 앵글 |
|--------|------|------|
| "안 받은 9원" | "토스에서 체크하면 바로 받아요." | 미수령 |
| "판다가 기다려요" | "토스에서 밥 주면 9원 받아요." | 감성 |
| "남은 미션 2개" | "토스에서 완료하면 9원 받아요." | 행동묘사 |

### 3. 활성 유저

**특성**: 정기적으로 앱을 사용 중. 앱을 잘 알고 있음.
**핵심 전략**: 새 콘텐츠 안내 + 심화 참여 유도
**소재 앵글**: 신규 컨텐츠, 일상루틴

| 타이틀 | 본문 | 앵글 |
|--------|------|------|
| "새 미션 추가" | "토스에서 바로 확인해보세요." | 업데이트 |
| "오늘 미션 열림" | "토스에서 체크 3번이면 9원 받아요." | 행동묘사 |
| "판다 새 옷 도착" | "토스에서 입혀보세요." | 감성·재미 |

### 4. 이탈 유저 (7일+ 미접속)

**특성**: 한동안 앱을 열지 않음. 밀린 보상이 쌓여 있을 수 있음.
**핵심 전략**: 밀린 보상 + 캐릭터 감성으로 재방문 유도
**소재 앵글**: 리텐션, 확실한 보상

| 타이틀 | 본문 | 앵글 |
|--------|------|------|
| "밀린 27원" | "토스에서 체크하면 받을 수 있어요." | 밀린 보상 |
| "판다가 울어요" | "토스에서 밥 주면 9원 받아요." | 감성 리텐션 |
| "놓친 미션 5개" | "토스에서 아직 도전할 수 있어요." | 미완료 |

## 캠페인 분리 원칙

### 1캠페인 1세그먼트 1랜딩URL

앱인토스 콘솔 제약: **하나의 캠페인에서는 소재별로 다른 랜딩 URL을 설정할 수 없다.**

따라서:
- 세그먼트가 다르면 → 캠페인을 분리
- 랜딩 페이지가 다르면 → 캠페인을 분리
- 같은 세그먼트 + 같은 랜딩이면 → 하나의 캠페인에 소재 여러 개

### 추천 캠페인 구조

```
캠페인 포트폴리오:

1. [상시] 신규 유저 획득 캠페인
   - 세그먼트: 전체 유저
   - 소재: 서비스 소개 앵글 5~7개
   - 랜딩: 앱 메인 페이지

2. [상시] 이탈 유저 재방문 캠페인
   - 세그먼트: 7일+ 미접속 유저
   - 소재: 밀린 보상 + 감성 앵글 5개
   - 랜딩: 앱 메인 페이지

3. [이벤트] 시즌 이벤트 캠페인
   - 세그먼트: 활성 유저
   - 소재: 이벤트 특화 5개
   - 랜딩: 이벤트 전용 페이지
   - 기간: 이벤트 기간만 운영
```

## 주의사항

1. **세그먼트를 너무 잘게 나누면** 각 세그먼트의 유저 수가 적어져 MAB 학습이 느려진다. 의미 있는 크기의 그룹으로 나눈다.
2. **같은 유저에게 여러 캠페인의 메시지가 동시에 갈 수 있다**. 세그먼트가 겹치는 경우를 고려한다.
3. **세그먼트 조건은 콘솔에서 설정**한다. 이 스킬에서는 소재만 생성하므로, 세그먼트 설정은 콘솔 가이드를 참고한다.
