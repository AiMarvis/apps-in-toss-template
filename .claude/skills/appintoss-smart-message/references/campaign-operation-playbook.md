# 캠페인 운영 플레이북

스마트 발송 캠페인의 전체 라이프사이클과 포트폴리오 전략.

---

## 캠페인 라이프사이클

### 전체 흐름

```
소재 설계 → 검토 요청 (2~3일) → 캠페인 활성화 → 성과 모니터링 → 복기 → 다음 세트
    ↑                                                                    ↓
    └──────────────────── 3~4주 주기 반복 ──────────────────────────────────┘
```

### 단계별 가이드

#### 1단계: 소재 설계 (Phase 1~3)

- SKILL.md의 Phase 1~3에 따라 소재 생성
- 최소 5세트, 권장 7~10세트
- 5앵글 다양성 확보 필수
- 반려 체크리스트 자가검증 완료

#### 2단계: 검토 요청 (영업일 2~3일)

- 콘솔에서 검토 요청 제출
- 영업일 기준 2~3일 소요
- **반려 시**: 수정 후 재제출 (추가 2~3일)
- **팁**: 이벤트 캠페인은 최소 1주 전에 제출

#### 3단계: 캠페인 활성화

- 검토 승인 후 캠페인 활성화
- MAB 알고리즘이 자동으로 소재 최적화 시작
- 초기 1~3일은 탐색 단계 (모든 소재에 골고루 노출)

#### 4단계: 성과 모니터링

- 활성화 후 3~7일에 첫 번째 성과 확인
- 콘솔에서 노출수/클릭수/CTR 확인
- Winner/Loser 패턴 파악 시작

#### 5단계: 복기 & 다음 세트 준비

- Phase 4(성과 복기 프레임워크) 실행
- Winner 인사이트 → 다음 세트에 반영
- 다음 세트를 설계하고 2~3일 전 검토 요청

## 캠페인 포트폴리오 전략

### 3종 캠페인 체계

하나의 앱에서 3가지 종류의 캠페인을 동시 운영하면 유저 라이프사이클 전체를 커버한다.

#### 캠페인 A: Always-On (상시 운영)

| 항목 | 값 |
|------|-----|
| 목적 | 신규 유저 획득 |
| 세그먼트 | 전체 유저 (미접속 포함) |
| 소재 앵글 | 서비스 소개 + 호기심 + 확실한 보상 |
| 소재 수 | 7~10개 |
| 기간 | 상시 (3~4주마다 소재 교체) |
| 랜딩 | 앱 메인 페이지 |

#### 캠페인 B: Event-Driven (이벤트)

| 항목 | 값 |
|------|-----|
| 목적 | 시즌 이벤트/업데이트 알림 |
| 세그먼트 | 활성 유저 또는 전체 |
| 소재 앵글 | 이벤트 특화 + 사회적 증거 |
| 소재 수 | 5~7개 |
| 기간 | 이벤트 기간 (검토 1주 전 제출) |
| 랜딩 | 이벤트 전용 페이지 |

#### 캠페인 C: Win-Back (리텐션)

| 항목 | 값 |
|------|-----|
| 목적 | 이탈 유저 재방문 |
| 세그먼트 | 7일+ 미접속 유저 |
| 소재 앵글 | 밀린 보상 + 캐릭터 감성 |
| 소재 수 | 5개 |
| 기간 | 상시 |
| 랜딩 | 앱 메인 페이지 |

## 소재 로테이션 주기

### 왜 교체가 필요한가

- 같은 소재를 4주 이상 사용하면 **소재 피로도** 발생
- 유저가 같은 메시지를 여러 번 보면 무시하게 됨
- MAB 알고리즘도 새 소재가 들어와야 새 최적점을 찾음

### 로테이션 스케줄

```
[3~4주 로테이션 예시]

Week 1: 소재 세트 A 활성화 (탐색 단계)
Week 2: 소재 세트 A 최적화 (활용 단계)
Week 3: 소재 세트 A 최적화 + 세트 B 검토 요청
Week 4: 세트 B 승인 대기 + 세트 A 성과 복기
Week 5: 세트 B 활성화 (세트 A 종료)
```

### Winner 소재 승계 원칙

세트 교체 시, 이전 세트의 Winner 소재를 그대로 가져올 수 있다:
- Winner의 **핵심 키워드는 유지** (성과가 검증된 요소)
- Winner의 **앵글을 교차** (같은 키워드를 다른 방식으로)
- 새 실험 소재 2~3개 추가 (새 가설 검증)

## 캠페인 타이틀 네이밍 컨벤션

콘솔에서 캠페인을 관리하기 쉽도록 이름 규칙을 잡는다.

**형식**: `[유형] [목적] [세트번호] (시작일)`

**예시**:
- `[상시] 신규획득 v3 (03.15)`
- `[이벤트] 벚꽃이벤트 v1 (04.01)`
- `[리텐션] 이탈복귀 v2 (03.20)`

## 성과 벤치마킹

### 일반적인 CTR 벤치마크 (앱인토스 스마트 발송)

| 성과 | CTR | 평가 |
|------|-----|------|
| 매우 높음 | 6%+ | 프로모션 있는 최적화된 소재 |
| 높음 | 4~6% | 좋은 소재 + 프로모션 |
| 보통 | 2~4% | 기본적인 소재 |
| 낮음 | 1~2% | 개선 필요 |
| 매우 낮음 | ~1% 미만 | 소재 전면 교체 필요 |

**참고**: 두바이 쫀득 쿠키 사례에서 9원 프로모션 + 최적 소재 = CTR 6.83%

### 성과가 안 나올 때 점검 순서

1. **소재 문제인가?** → 앵글 다양성 부족, 보상 불명확, 서비스 이해 부족
2. **세그먼트 문제인가?** → 타겟이 너무 넓거나 좁음
3. **프로모션 문제인가?** → 금액 미명시, "포인트" 대신 "원" 사용 여부
4. **소재 피로도인가?** → 4주 이상 같은 소재 사용 중
5. **알고리즘 학습 부족인가?** → 소재 수 2~3개, 캠페인 기간 3일 미만
